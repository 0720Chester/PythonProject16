## 实验原理

### LBS 线性混合蒙皮

LBS 是一种经典的骨骼动画技术，其核心思想是：每个顶点的最终位置由多个关节的刚体变换加权求和得到。

### 四个阶段详解

**阶段 (a)：模板网格与蒙皮权重**

初始状态为 T-pose 模板人体网格。每个顶点都带有一组对各关节的影响权重。如果某个顶点更靠近手臂，那么它通常会更受肩、肘、腕等关节影响。

这一步的重点是理解：网格还没根据人物体型改变，网格也还没根据姿态弯曲，但每个顶点已经知道将来应该主要跟着哪些骨骼走。

在 LBS 实现中，最终每个顶点的变换矩阵，就是由蒙皮权重对各关节变换矩阵加权得到的。

**阶段 (b)：形状校正与关节回归**

形状参数控制这个人长什么样。例如高矮、胖瘦、肩宽、腿长等，都可以由形状空间中的若干系数表示。

形状校正后得到改变了体型的网格，然后再根据这个网格利用关节回归器得到关节位置。关节位置不是固定常数，而是由形状后的网格回归出来的。

**阶段 (c)：姿态相关校正**

蒙皮并非把骨骼旋转一下、皮肤跟着转这么简单，因为人体在弯曲时，肩膀、肘部、膝盖附近会出现额外的几何变化，仅靠骨骼刚体旋转无法表达。所以 SMPL 在进入真正的 LBS 前，还会加入一项姿态混合变形。

实现思路是先把姿态参数转成旋转矩阵，然后通过姿态主成分线性映射得到姿态偏移量，并加到形状校正后的网格上。

**阶段 (d)：线性混合蒙皮**

经过上述步骤，已经考虑了形状的关节位置、姿态校正的顶点、每个顶点对各关节的权重，之后进入真正的 LBS。

每个顶点根据其受各关节影响的权重，以及各关节在运动学链上的全局刚体变换，加权计算得到最终位置。

## 项目结构

## 快速开始

### 参数说明

--model-path 参数指定 SMPL 模型文件的路径，类型为字符串，默认值为项目目录下的 SMPL_NEUTRAL.pkl。

--out-dir 参数指定输出目录，类型为字符串，默认值为 ./outputs。

--joint-id 参数指定要可视化权重的关节编号，类型为整数，取值范围 0 到 23，默认值为 18。

--num-betas 参数指定使用的形状参数维度，类型为整数，默认值为 10。

### 查看结果

程序运行完成后，所有可视化结果保存在 outputs 目录下。打开各 PNG 文件即可查看 LBS 各阶段的可视化效果，打开 summary.txt 可查看详细的实验数据和结论。

## 技术实现

### 核心算法流程

程序首先加载 SMPL 模型文件，提取模板网格、形状主成分、姿态主成分、关节回归器和蒙皮权重等组件。

然后构建演示用的形状参数和姿态参数。形状参数控制体型特征，姿态参数控制各关节的旋转角度。

接着进入 LBS 四个阶段的计算：先获取模板网格和蒙皮权重，再通过形状参数进行形状校正并回归关节位置，然后通过姿态参数进行姿态校正，最后通过蒙皮权重和关节变换完成 LBS 蒙皮。

### 关键技术点

Chumpy 兼容性处理：使用自定义 Shim 类替代 chumpy，解决 Python 高版本的兼容问题。

Posedirs 维度处理：正确理解并处理三维 posedirs 张量，使用 einstein 求和计算姿态偏移。

OpenMP 冲突解决：设置环境变量避免多个 OpenMP 运行时库的冲突。

坐标系统转换：将 SMPL 坐标系转换到适合可视化的坐标系。

## 输出说明

### stage_a_template_weights.png

展示 T-pose 模板网格，热力图颜色表示指定关节的蒙皮权重分布。颜色越亮表示该关节对该顶点的影响越大。

### stage_b_shaped_joints.png

展示形状校正后的网格，白色点表示回归得到的关节位置。可以看到关节位置随体型变化而自动调整。

### stage_c_pose_offsets.png

展示姿态校正后的网格，颜色表示姿态偏移量的大小。可以看到偏移主要集中在关节弯曲处，如肘部、膝盖等位置。

### stage_d_lbs_result.png

展示最终的 LBS 蒙皮结果，白色点为变换后的关节位置。可以看到完整的人体姿态，关节处的变形更加自然。

### comparison_grid.png

将四个阶段以 2x2 布局排列，标题清楚标出各阶段名称，便于对比观察 LBS 全流程中各阶段的变化。

### all_joint_weights.png

展示所有关节在人体表面的主导控制区域，不同颜色表示不同的主导关节。可以清晰看到各关节的影响范围。

### summary.txt

记录模型的顶点数、面片数、关节数等基本信息，以及实验使用的形状参数和姿态参数，最后给出实验结论。

## 常见问题

### Python 版本兼容问题

如果遇到 inspect.getargspec 相关错误，说明 chumpy 库与当前 Python 版本不兼容。程序已内置自动修复功能，无需手动处理。

如果提示模型文件不存在，请确认已从 SMPL 官网下载模型文件并放置在正确路径，或使用 --model-path 参数指定正确路径。
