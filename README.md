## 项目简介

本项目基于 SMPL 模型，完整实现了 LBS 蒙皮过程的可视化。通过手写 LBS 实现，将蒙皮过程的四个关键阶段进行可视化展示，帮助理解参数化人体模型中模板网格、形状参数、姿态参数、关节回归器和蒙皮权重之间的关系。

## 实验目标

理解 SMPL 模型组件关系，掌握模板网格、形状参数、姿态参数、关节回归器和蒙皮权重之间的数学关系。理解 LBS 四个阶段：模板网格与蒙皮权重、形状校正与关节回归、姿态相关校正、线性混合蒙皮最终结果。学会调用 SMPL 模型，把官方 LBS 实现中的关键中间量单独提取出来做可视化。

## 实验原理

LBS 是一种经典的骨骼动画技术，其核心思想是每个顶点的最终位置由多个关节的刚体变换加权求和得到。

阶段 (a) 模板网格与蒙皮权重。初始状态为 T-pose 模板人体网格，每个顶点都带有一组对各关节的影响权重。如果某个顶点更靠近手臂，那么它通常会更受肩、肘、腕等关节影响。这一步的重点是理解网格还没根据人物体型改变，网格也还没根据姿态弯曲，但每个顶点已经知道将来应该主要跟着哪些骨骼走。在 LBS 实现中，最终每个顶点的变换矩阵就是由蒙皮权重对各关节变换矩阵加权得到的。

阶段 (b) 形状校正与关节回归。形状参数控制这个人长什么样，例如高矮、胖瘦、肩宽、腿长等，都可以由形状空间中的若干系数表示。形状校正后得到改变了体型的网格，然后再根据这个网格利用关节回归器得到关节位置。关节位置不是固定常数，而是由形状后的网格回归出来的。

阶段 (c) 姿态相关校正。蒙皮并非把骨骼旋转一下皮肤跟着转这么简单，因为人体在弯曲时，肩膀、肘部、膝盖附近会出现额外的几何变化，仅靠骨骼刚体旋转无法表达。所以 SMPL 在进入真正的 LBS 前，还会加入一项姿态混合变形。实现思路是先把姿态参数转成旋转矩阵，然后通过姿态主成分线性映射得到姿态偏移量，并加到形状校正后的网格上。

阶段 (d) 线性混合蒙皮。经过上述步骤，已经考虑了形状的关节位置、姿态校正的顶点、每个顶点对各关节的权重，之后进入真正的 LBS。每个顶点根据其受各关节影响的权重，以及各关节在运动学链上的全局刚体变换，加权计算得到最终位置。

## 环境要求

需要安装依赖库：torch、numpy、matplotlib、smplx、scipy。推荐使用 Python 3.8 或更高版本。

需要 SMPL 模型文件 SMPL_NEUTRAL.pkl，可从 SMPL 官网 https://smpl.is.tue.mpg.de/ 下载，放置在项目根目录即可。

## 项目结构

main.py 是主程序文件。SMPL_NEUTRAL.pkl 是模型文件，需自行下载。outputs 是输出目录，包含以下文件：stage_a_template_weights.png 展示模板网格和蒙皮权重热力图，stage_b_shaped_joints.png 展示形状校正后的网格和关节位置，stage_c_pose_offsets.png 展示姿态校正偏移量，stage_d_lbs_result.png 展示最终 LBS 蒙皮结果，comparison_grid.png 是四个阶段对比图，all_joint_weights.png 展示所有关节权重分布，summary.txt 记录实验总结。

## 快速开始

使用默认参数运行：python main.py。自定义参数运行：python main.py --model-path 模型路径 --out-dir 输出目录 --joint-id 关节编号 --num-betas 形状参数维度。程序运行完成后在 outputs 目录查看结果。

## 技术实现

程序加载 SMPL 模型后提取各组件，构建形状参数和姿态参数，依次执行 LBS 四个阶段。使用自定义 Shim 类解决 chumpy 兼容问题，使用 einstein 求和计算姿态偏移，设置环境变量解决 OpenMP 冲突。

## 输出说明

各阶段图片展示 LBS 流程的不同状态，summary.txt 记录模型信息和实验数据。

## 常见问题

chumpy 兼容问题已内置自动修复。OpenMP 冲突已通过环境变量解决。模型文件缺失请确认下载并放置正确。
